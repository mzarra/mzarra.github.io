seSales v1.4 Beta 1 Available

 

Zarra Studios released Beta 1 of their upcoming version 1.4 release of seSales.  

Download Now!

Several major enhancements are included in this release.  However, this is a beta release and a few things should be noted:

This release is not “feature complete” but is “data complete”.  What this means is that while some features are not completed yet, all of the required data and data entry is complete.  Barring any unforeseen situations, there will not be another data migration for this release prior to the final release.  As the beta progresses the additional features required and expected will be added.  However we wanted to get this release out into your hands to confirm that all of the data that is needed is in place.

With that said, listed below are the current release notes.  These notes will be updated with subsequent releases:

Completed Features:

  1. •Added the ability to handle rental items which includes the ability to print rental agreements directly on the invoice and track rental items by a unique identifier such as a serial number.  Rental items can have multiple rental rates defined for them and will automatically select the appropriate rental rate.

  2. •Added universal key bindings.  Any menu item can now have a key binding assigned to it.

  3. •Added a new customizable 80mm receipt printer format.  Any paper size that is under 4" wide will automatically use the 80mm format.

  4. •Added a MYOB compatible invoice/sales export.

  5. •Added MYOB compatible inventory export.

  6. •Added MYOB compatible customer export.

  7. •Added check to ensure the company data file is writable.

  8. •Added ability to flag a customer and/or invoice as tax exempt.

  9. •Added ability for company administrators to edit the date of an invoice.

  10. •Added additional data validation on Item Management.

  11. •Added additional data validation on Vendor Management.

  12. •Added additional data validation on Customer Management.

  13. •Increased the frequency of background saves to the company file while doing data entry in the Item, Vendor and Customer Management windows.

  14. •Added ability for the Item Management window to detect when the productID pattern changes and update the pattern for future items added.

  15. •Added defaulting of category and taxes to the settings used on the last item added to facilitate data input.

  16. •Added productID to inventory export.

  17. •Updated the Print Copy and moved it to the File Menu to be more consistent with the Apple UI Guidelines.

  18. •Added default to automatically close an invoice's window once the invoice has been posted.

  19. •Removed Invoice Posting confirmation.  Canceling out of the print window will now cancel the posting of an invoice.

  20. •Added the ability to force the creation of a drawer prior to creating an invoice.

  21. •Added ability to open a different company file instead of migrating data in the current company file.

Localizations:

  1. •Added Canadian Localization.

  2. •Added Great Britain Localization.

Bug Fixes:

  1. •Corrected an issue in the audit creation that was preventing the ability create new inventory audits.

  2. •Corrected an issue with the syncing of new customers with Address Book.

  3. •Altered the creation of new items in the item management window to default to a quantity of 1.

  4. •Corrected an issue when canceling out of adding a credit payment to a customer’s account.

  5. •Corrected keyboard flow on PayPal payment entry sheet.

  6. •Removed the initial registration dialog window pop-up on Company loading.

  7. •Corrected the display of payment dates on invoices.

  8. •Corrected issue with display of archived discounts.

  9. •Corrected display issues on the Discount Management window.

  10. •Corrected an issue with the calculations used in the print invoice.  It is now possible to rearrange the columns: description, quantity and cost.

Note:

This release will perform a database migration on your company data file.  The company data file will be copied to a backup prior to the migration.  As with any data migration it is possible an error will occur.  Please back up your company file before proceeding with the data migration.

If you choose to download and work with this beta please send all comments, bugs, issues support@zarrastudios.com.  As always, your support and review of this beta will ensure that all of the issues are identified and corrected prior to the final release.

MSZ
